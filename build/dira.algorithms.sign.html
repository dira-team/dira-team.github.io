detect_sign(img)[source]¶

Detect and classify a traffic sign in the input image.

Parameters

img – numpy RGB image, not normalized (range 0-255)

Returns

2 values, including: ___1. int sign type:

0: SIGN_NONE 1: SIGN_TURN_RIGHT 2: SIGN_TURN_LEFT

___2. a tuple specifying sign position and size (x, y, w, h), in which (x,y) is the coordinate of the top left pixel of the traffic sign, w is width and h is height.

>>> detect_sign(rgb_frame)

Input image with a different colour space from RGB might give inaccurate result.

find_by_color(img_np)[source]¶

Find objects in the input image based on color.

Parameters

  • img_np – numpy RGB image, not normalized (range 0-255)

  • hsv_color_ranges – sequence of color ranges to be detected. Each range is comprised of lower bound and upper bound.

  • min_saturation – the minimum saturation to be detected.

  • min_value – the minimum value (or brightness) to be detected.

  • min_sign_size – the minimum size (in pixel) of traffic sign to be detected.

  • differential_rate – the maximum differential rate between height and width of the traffic sign to be detected. This rate is computed using the formula: |width-height| / [(width+height)/2]

Returns

2 lists: ___1. List of numpy rgb images of colour-matched regions. ___2. List of the respectively region’s position and size. Each element of the list is a tuple (x,y,w,h), in which (x,y) is the coordinate of the top left pixel of the region, w is width and h is height.

>>> find_by_color(rgb_frame)
(np.array, np.array), ((60,60,100,100), (80,80,30,30), )

Input image with a different colour space from RGB might give bad result.
